
## 关键问题

| 字段 | 值 | 分析 |
|------|-----|------|
| type | `range` | 索引范围扫描（phone_prefix + name LIKE） |
| key | `idx_prefix_name` | 用了联合索引 |
| rows | ~17,682 | 预估扫描约 1.8 万行 |
| Extra | **`Using where`** | WHERE 条件在**回表后**才过滤 |

## 为什么慢

关闭 ICP 后，MySQL 只用索引定位 `phone_prefix = '1380'` 匹配的行，然后**全部回表**读取完整行数据，再在 server 层用 `name LIKE '张%'` 过滤。

如果 `phone_prefix = '1380'` 匹配 4 万行，但 `name LIKE '张%'` 只有 2000 行匹配，那么 **3.8 万次回表是无效的**。

实际耗时：约 **120 ms**。
