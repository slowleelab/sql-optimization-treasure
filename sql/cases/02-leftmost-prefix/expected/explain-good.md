| id | select_type | table           | partitions | type | possible_keys          | key                    | key_len | ref               | rows | filtered | Extra                 |

## 关键改进

| 字段 | 值 | 分析 |
|------|-----|------|
| type | `ref` | 索引等值查找 |
| key | `idx_user_status_created` | 联合索引被正确使用 |
| key_len | `18` | user_id(8) + status(1) + created_at(部分范围) 三列都参与了索引 |
| rows | `3` | 精确定位到 3 行（seed 中的固定测试数据） |
| Extra | `Using index condition` | 索引条件下推，在索引层完成过滤 |

## 为什么快

补全 `user_id = 12345` 后，查询满足最左前缀原则：
1. 通过 `user_id=12345` 在 B+ 树中精确定位索引入口
2. 在该范围内 `status=1` 进一步缩小范围
3. `created_at > '2026-01-01'` 做范围扫描

三列全部参与索引过滤，从 30 万行扫描缩减到 3 行，耗时 < 1ms。
