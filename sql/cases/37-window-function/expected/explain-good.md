# EXPLAIN 参考结果 - good.sql（ROW_NUMBER 窗口函数取每部门最高薪资）

## MySQL 8.0（实测 8.0.46，10 万行薪资，100 个部门）

查询每个部门薪资最高的员工（结果约 100 行，每部门 1 人）。

```
+----+-------------+------------+--------+------------------+------------------+---------+------+--------+----------+----------------+
| id | select_type | table      | type   | possible_keys     | key              | key_len | ref  | rows   | filtered | Extra          |
+----+-------------+------------+--------+------------------+------------------+---------+------+--------+----------+----------------+
|  1 | PRIMARY     | <derived2> | ALL    | NULL             | NULL             | NULL    | NULL |  99876 |   100.00 | Using filesort |
|  2 | DERIVED     | t_salary   | index  | idx_dept_salary  | idx_dept_salary  | 88      | NULL |  99876 |   100.00 | Using index    |
+----+-------------+------------+--------+------------------+------------------+---------+------+--------+----------+----------------+
```

## 关键改进

| 字段 | 值 | 分析 |
|------|-----|------|
| 步骤数 | **2 行** | 仅 2 步（bad 方案需相关子查询反复执行） |
| type (id=2) | **`index`** | **索引全扫描**，走 idx_dept_salary |
| key (id=2) | `idx_dept_salary` | 利用 (dept, salary) 联合索引 |
| Extra (id=2) | **`Using index`** | **覆盖索引！** 不回表 |
| rows (id=2) | ~99,876 | 单次扫描全部行（仅一次，非 10 万次） |
| 子查询次数 | **0** | 无相关子查询，无嵌套循环 |

## 为什么快

窗口函数将"每部门取最高薪资"转化为单次扫描 + 分组编号：

1. **单次索引扫描**：内层查询走 `idx_dept_salary` 索引全扫描（type=index）
2. **覆盖索引**：dept、salary、id 都在索引中（联合索引 + 主键），`Using index` 不回表
3. **索引有序**：idx_dept_salary 已按 (dept, salary) 排序，窗口函数直接利用此顺序分区编号
4. **ROW_NUMBER 编号**：按 dept 分区、salary DESC 编号 1,2,3...
5. **外层过滤**：从派生表取 rn = 1 的行（每部门薪资最高的）

对比 bad 方案：
- bad：10 万次相关子查询，每次扫约 1000 行 = 累计约 1 亿次索引行读取
- good：**1 次**索引全扫描，覆盖索引不回表 = 10 万次索引顺序读

窗口函数消除了嵌套循环，将 O(N*M) 降为 O(N log N)（排序分区）。且因索引已有序，排序开销也被优化器消除。

实际耗时：约 **70 ms**（单次覆盖索引扫描 + 窗口函数计算）。

## 量化对比

| 指标 | bad.sql (相关子查询) | good.sql (窗口函数) | 提升 |
|------|---------------------|---------------------|------|
| 执行步骤 | 2（但内层重复 10 万次） | 2（各执行 1 次） | 消除重复执行 |
| 外层 type | ALL (全表扫描) | index (索引扫描) | 走索引 |
| 子查询执行次数 | ~100,000 | 0 | **完全消除** |
| 索引行读取 | ~100,000,000 | ~99,876 (顺序读) | **减少 99.9%** |
| 回表 | 是 (子查询回表) | 否 (覆盖索引) | **消除回表** |
| 耗时 | 1350 ms | 70 ms | **约 19 倍** |

::: tip 窗口函数的常见用法
1. **ROW_NUMBER()**：无并列排名，取每组 Top N 最常用
2. **RANK() / DENSE_RANK()**：有并列时的排名（跳号 / 不跳号）
3. **LAG() / LEAD()**：取前/后 N 行的值（如环比、同比）
4. **SUM() OVER()**：累计求和、移动平均
5. **FIRST_VALUE() / LAST_VALUE()**：取分组内首/末值

8.0 之前这类需求只能用相关子查询、自连接或存储过程，性能和可读性都差。窗口函数是 8.0 最实用的新特性之一。
:::
