# EXPLAIN 参考结果 - good.sql（ROW_NUMBER 窗口函数取每部门最高薪资）

## MySQL 8.0（实测 8.0.46，10 万行薪资，100 个部门）

查询每个部门薪资最高的员工（结果约 100 行，每部门 1 人）。

```
+----+-------------+------------+--------+------------------+------------------+---------+------+--------+----------+---------------+
| id | select_type | table      | type   | possible_keys     | key              | key_len | ref  | rows   | filtered | Extra         |
+----+-------------+------------+--------+------------------+------------------+---------+------+--------+----------+---------------+
|  1 | PRIMARY     | <derived2> | ref    | <auto_key0>      | <auto_key0>      | 8       | const| 10     |   100.00 | Using filesort|
|  2 | DERIVED     | t_salary   | ALL    | NULL             | NULL             | NULL    | NULL | 100000 |   100.00 | Using filesort|
+----+-------------+------------+--------+------------------+------------------+---------+------+--------+----------+---------------+
```

## 关键改进

| 字段 | 值 | 分析 |
|------|-----|------|
| 步骤数 | **2 行** | 仅 2 步（bad 方案需相关子查询反复执行） |
| type (id=2) | `ALL` | 内层全表扫描（仅一次） |
| rows (id=2) | ~100,000 | 单次扫描全部行（仅一次，非 10 万次） |
| 子查询执行次数 | **0** | 无相关子查询，无嵌套循环 |
| Extra (id=2) | `Using filesort` | 窗口函数需排序分区（单次排序） |

## 为什么快

窗口函数将"每部门取最高薪资"转化为单次扫描 + 分组排序：

1. **单次全表扫描**：内层查询扫描 t_salary 全部 10 万行（仅 1 次）
2. **窗口排序**：按 dept 分区、salary DESC 排序，为每行分配 ROW_NUMBER
3. **物化派生表**：排序结果存入派生表（`<derived2>`）
4. **外层过滤**：从派生表取 rn = 1 的行（每部门薪资最高的）

对比 bad 方案：
- bad：10 万次相关子查询，每次扫约 1000 行 = 累计约 1 亿次索引行读取
- good：**1 次**全表扫描 + 1 次排序 = 10 万行顺序读 + 排序

窗口函数消除了嵌套循环，将 O(N*M) 降为 O(N log N)（排序分区）。

实际耗时：约 **247 ms**（单次全表扫描 + 窗口排序）。

注：虽然内层查询未使用 `idx_dept_salary` 索引（type=ALL 而非 index），但单次全表扫描 + 排序的代价远低于 10 万次相关子查询。优化器判断全表扫描后内存排序比走索引再回表更高效。

## 量化对比

| 指标 | bad.sql (相关子查询) | good.sql (窗口函数) | 提升 |
|------|---------------------|---------------------|------|
| 执行步骤 | 2（但内层重复 10 万次） | 2（各执行 1 次） | 消除重复执行 |
| 外层 type | ALL (全表扫描) | ALL (全表扫描) | 相同 |
| 子查询执行次数 | ~100,000 | 0 | **完全消除** |
| 索引行读取 | ~100,000,000 | ~100,000 (单次扫描) | **减少 99.9%** |
| 耗时 | 17,560 ms | 247 ms | **约 71 倍** |

::: tip 窗口函数的常见用法
1. **ROW_NUMBER()**：无并列排名，取每组 Top N 最常用
2. **RANK() / DENSE_RANK()**：有并列时的排名（跳号 / 不跳号）
3. **LAG() / LEAD()**：取前/后 N 行的值（如环比、同比）
4. **SUM() OVER()**：累计求和、移动平均
5. **FIRST_VALUE() / LAST_VALUE()**：取分组内首/末值

8.0 之前这类需求只能用相关子查询、自连接或存储过程，性能和可读性都差。窗口函数是 8.0 最实用的新特性之一。
:::
