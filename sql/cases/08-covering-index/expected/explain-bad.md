```

| id | select_type | table     | type | possible_keys     | key               | key_len | ref   | rows | filtered | Extra |
+----+-------------+-----------+------+-------------------+-------------------+---------+-------+------+----------+-------+
|  1 | SIMPLE      | t_product | ref  | idx_category_price| idx_category_price| 4       | const | 2987 |   100.00 | NULL  |

```

## 关键问题

| 字段 | 值 | 分析 |
|------|-----|------|
| type | `ref` | 走了索引（这部分没问题） |
| key | `idx_category_price` | 用了索引 |
| rows | 2987 | 扫描约 3000 行 |
| Extra | NULL | **没有 Using index** -> 需要回表 |

## 为什么慢

虽然 `WHERE category_id = 50` 走了索引定位，但 `SELECT *` 要求返回所有字段（包括 `description` 这个 TEXT 长文本字段）。

`idx_category_price (category_id, price)` 索引中不包含 `description`、`stock`、`name` 等字段，因此 MySQL 必须：

1. 从索引中找到 2987 条匹配的主键 id
2. **逐条回表**到聚簇索引（主键索引）读取完整行数据
3. 其中 `description` 是 TEXT 类型，可能存储在溢出页（off-page），产生额外 I/O

2987 次回表 = 2987 次随机 I/O，这是性能瓶颈。
